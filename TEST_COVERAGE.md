# Таблица сценариев выставления заявок

Все возможные комбинации параметров заявки и ожидаемые результаты.

---

## 1. Тип инструмента

| Инструмент | Код класса | Пример | Особенности |
|------------|------------|--------|-------------|
| Акция | TQBR | GAZP, SBER | Стандартная логика |
| Облигация ОФЗ | TQOB | SU26245RMFS9 | edge = LimitActuationOrderBondEdge |
| Облигация корп. | TQCB, EQOB, TQIR, TQRD | RU000A10BFF4 | edge = LimitActuationOrderBondEdge |

---

## 2. Операция

| Операция | Код | Проверки |
|----------|-----|----------|
| Покупка | B | PRICEMIN, объём, срабатывание, цена облигации, средняя цена |
| Продажа | S | Наличие позиции, PRICEMIN не проверяется |

---

## 3. Цена заявки (покупка)

| Сценарий | Цена | PRICEMIN | Ожидаемый результат |
|----------|------|----------|---------------------|
| Цена > PRICEMIN | 1000 | 800 | ✅ Принята |
| Цена = PRICEMIN | 800 | 800 | ✅ Принята |
| Цена < PRICEMIN | 500 | 800 | ❌ Отклонена: price below PRICEMIN |
| Цена = 0 | 0 | 800 | → Конвертируется в min_price_step (0.1) → ❌ Отклонена |
| Цена < 0 | -100 | 800 | ❌ Отклонена: Invalid |

---

## 4. Цена заявки (продажа)

| Сценарий | Позиция | Ожидаемый результат |
|----------|---------|---------------------|
| Есть позиция | qty >= заявки | ✅ Принята |
| Нет позиции | 0 | ❌ Отклонена: insufficient position |
| Позиция < заявки | 5 < 10 | ❌ Отклонена: insufficient position |
| Цена = 0 | Есть | ✅ Принята (PRICEMIN не проверяется) |
| Цена < 0 | Есть | ✅ Принята (PRICEMIN не проверяется) |

---

## 5. Количество

| Сценарий | Количество | Ожидаемый результат |
|----------|------------|---------------------|
| Валидное | 10 | ✅ Принята |
| = 0 | 0 | ❌ Отклонена: Invalid |
| < 0 | -5 | ❌ Отклонена: Invalid |
| = 1 | 1 | ✅ Принята |

---

## 6. Объём заявки (покупка)

| Сценарий | Объём | Limit | Ожидаемый результат |
|----------|-------|-------|---------------------|
| Объём < лимита | 50000 | 200000 | ✅ Принята |
| Объём = лимиту | 200000 | 200000 | ✅ Принята |
| Объём > лимита | 250000 | 200000 | ❌ Отклонена: exceeds limit |
| Объём = 0 | 0 | 200000 | ❌ Отклонена: Invalid (qty=0) |
| Объём < 0 | -1000 | 200000 | ❌ Отклонена: Invalid (qty<0) |

**Расчёт объёма:** `Количество × Цена × Лот`

---

## 7. Срабатывание (actuation)

**Формула:** `(LAST - Цена_заявки) / Цена_заявки × 100%`

| Сценарий | LAST | Цена | Actuation | Edge | Результат |
|----------|------|------|-----------|------|-----------|
| Высокое срабатывание | 1000 | 900 | 11.1% | 5% | ✅ Принята (11.1% > 5%) |
| Низкое срабатывание | 300 | 290 | 3.4% | 5% | ❌ Отклонена: actuation below limit |
| Равно edge | 1000 | 952 | 5.0% | 5% | ❌ Отклонена (actuation < limit) |
| Edge = 0 (отключена) | 1000 | 500 | 100% | 0% | ✅ Принята (проверка отключена) |
| Продажа | — | — | — | — | ✅ Не проверяется |
| Тикер из исключений | — | — | — | — | ✅ Не проверяется |

**Исключения из проверки срабатывания:**
ENPG, RTKM, MTSS, NKNCP, UPRO, MGTSP, IRAO, MAGN, TGKA, GAZP, AFLT, ELFV, SMLT, SNGS, ALRS, MGNT, HYDR, VTBR, FEES, MVID, SGZH, AQUA, STSB, IVAT, VKCO

## 11. Источник цены

| Сценарий | UseFileParams | Метод определения | Результат |
|----------|---------------|-------------------|-----------|
| Цена из файла | true | Фиксированная из CSV | Цена не корректируется |
| Edge (покупка) | false | PriceAdjuster: LAST - 10 шагов | Цена корректируется вниз |
| Edge (продажа) | false | PriceAdjuster: LAST + 10 шагов | Цена корректируется вверх |
| Цена < PRICEMIN | false | PriceAdjuster: ставит PRICEMIN | Цена = PRICEMIN |
